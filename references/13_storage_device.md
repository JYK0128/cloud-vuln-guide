## 공통 점검 항목

### ST-01. 관리자 계정 및 패스워드 관리

- **위험도**: 상
- **점검 내용**: 스토리지 장비 관리자 기본 계정 및 패스워드 변경 여부
- **점검 방법**:
  - 관리 콘솔에서 계정 목록 확인
  - 기본 계정(admin/admin, root/password 등)으로 로그인 시도
- **양호 기준**: 기본 계정 삭제 또는 패스워드 변경, 강력한 패스워드 설정
- **취약 기준**: 기본 계정·패스워드 유지

### ST-02. 관리 인터페이스 접근 제한

- **위험도**: 상
- **점검 내용**: 스토리지 관리 인터페이스(웹 콘솔, SSH, CLI) 허가된 IP 제한
- **점검 방법**:
  - 관리 포트 접근 ACL 설정 확인
  - 외부 IP에서 관리 포트 접근 가능 여부 테스트
- **양호 기준**: 관리자 IP 대역만 관리 인터페이스 접근 허용
- **취약 기준**: 인터넷 또는 불특정 IP에서 관리 접속 허용

### ST-03. 암호화 설정

- **위험도**: 상
- **점검 내용**: 저장 데이터 암호화(at-rest encryption) 설정 여부
- **점검 방법**:
  - 스토리지 관리 콘솔에서 암호화 설정 확인
- **양호 기준**: 볼륨/LUN/공유 폴더 암호화 적용
- **취약 기준**: 암호화 미적용 (도난·분리 시 데이터 노출)

### ST-04. 데이터 전송 암호화

- **위험도**: 상
- **점검 내용**: 관리 인터페이스 및 데이터 전송 TLS/IPsec 사용 여부
- **양호 기준**: HTTPS, TLS, 또는 IPsec으로 전송 암호화
- **취약 기준**: HTTP, FTP 등 평문 프로토콜 사용

### ST-05. 접근 로그 및 감사

- **위험도**: 중
- **점검 내용**: 스토리지 접근 이벤트 로그 기록 및 보관 여부
- **양호 기준**: 접근 로그 90일 이상 보관, 외부 Syslog 전송
- **취약 기준**: 로그 미기록 또는 단기 보관

### ST-06. 패치 및 펌웨어 업데이트

- **위험도**: 상
- **점검 내용**: 스토리지 펌웨어 최신 보안 패치 적용 여부
- **점검 방법**:
  - 현재 펌웨어 버전 확인
  - 벤더사 보안 공지 비교
- **양호 기준**: 최신 보안 패치 적용
- **취약 기준**: 오래된 펌웨어 사용 (알려진 CVE 존재)

## SAN 전용 점검 항목

### SAN-01. 존(Zone) 설정 및 관리

- **위험도**: 상
- **점검 내용**: SAN 패브릭에서 호스트와 스토리지 LUN 간 존 설정(Zone 분리) 적절성
- **점검 방법**:
  - Fibre Channel 스위치에서 Zone 목록 확인
  - `show zone` 또는 관리 콘솔에서 Zone 구성 확인
- **양호 기준**: 서버별 전용 Zone 설정, 단일 initiator-target Zone
- **취약 기준**: 모든 포트가 동일 Zone에 포함 (All-Zone)

### SAN-02. LUN 마스킹 설정

- **위험도**: 상
- **점검 내용**: LUN 마스킹으로 허가된 호스트만 스토리지 LUN 접근 허용 여부
- **점검 방법**:
  - 스토리지 관리 콘솔에서 LUN 매핑 및 마스킹 설정 확인
- **양호 기준**: 특정 호스트 WWN에만 LUN 마스킹 적용
- **취약 기준**: LUN 마스킹 없음 또는 모든 호스트에 LUN 노출

### SAN-03. SAN 관리 네트워크 분리

- **위험도**: 중
- **점검 내용**: SAN 패브릭(관리 인터페이스)을 별도 관리 네트워크로 분리 여부
- **양호 기준**: 관리 트래픽과 SAN 데이터 트래픽 네트워크 분리
- **취약 기준**: 관리 인터페이스가 데이터 네트워크에 연결

### SAN-04. FC 스위치 계정 관리

- **위험도**: 상
- **점검 내용**: Fibre Channel 스위치 기본 계정 및 패스워드 변경 여부
- **양호 기준**: 기본 계정 삭제, 역할 기반 접근 제어(RBAC) 적용
- **취약 기준**: 기본 admin/password 유지

## NAS 전용 점검 항목

### NAS-01. 공유 폴더 접근 권한 최소화

- **위험도**: 상
- **점검 내용**: NAS 공유 폴더에 대한 과도한 권한 부여 여부
- **점검 방법**:
  - NAS 관리 콘솔에서 공유 폴더 권한 목록 확인
- **양호 기준**: 업무상 필요한 사용자/그룹에만 최소 권한 부여
- **취약 기준**: Everyone 읽기/쓰기 허용, 익명 접근 허용

### NAS-02. NFS/SMB 보안 설정

- **위험도**: 상
- **점검 내용**: NFS v2/v3 (인증 없음) 또는 SMBv1 (취약) 사용 여부
- **점검 방법**:
  - NAS 관리 콘솔에서 프로토콜 버전 확인

  ```bash
  # 클라이언트에서
  nmap -sV -p 2049,445 <nas-ip>
  ```

- **양호 기준**: NFSv4 (Kerberos 인증) 또는 SMBv3 이상 사용
- **취약 기준**: NFSv2/v3 익명 허용 또는 SMBv1 사용

### NAS-03. 스냅샷 및 백업 보호

- **위험도**: 중
- **점검 내용**: 스냅샷 및 백업 데이터에 대한 접근 통제 및 변조 방지
- **양호 기준**: 스냅샷 접근 권한 최소화, 불변(Immutable) 백업 설정
- **취약 기준**: 스냅샷을 누구나 삭제 가능

### NAS-04. 원격 접속 서비스 제한

- **위험도**: 중
- **점검 내용**: NAS의 SSH, FTP, Telnet 서비스 활성화 여부
- **점검 방법**:
  - NAS 관리 콘솔에서 실행 중인 서비스 확인
  - `nmap -sV <nas-ip>`
- **양호 기준**: 불필요한 서비스 비활성화, FTP 대신 SFTP 사용
- **취약 기준**: Telnet 또는 FTP 활성화

## 오브젝트 스토리지

### OBJ-01. 버킷/컨테이너 공개 접근 차단

- **위험도**: 상
- **점검 내용**: 오브젝트 스토리지 버킷이 공개(Public)로 설정되어 있는지 여부
- **점검 방법**:

  ```bash
  # AWS S3 예시
  aws s3api get-bucket-acl --bucket <bucket-name>
  aws s3api get-bucket-policy --bucket <bucket-name>
  # OpenStack Swift 예시
  swift stat <container-name>
  ```

- **양호 기준**: 버킷 공개 접근 차단, 정책으로 허가된 IAM/계정만 접근
- **취약 기준**: `AllUsers` 또는 `AuthenticatedUsers`에게 읽기/쓰기 허용

### OBJ-02. 버킷 버전 관리 및 MFA 삭제 보호

- **위험도**: 중
- **점검 내용**: 오브젝트 삭제/덮어쓰기 방지를 위한 버전 관리 및 삭제 보호 설정
- **양호 기준**: 버전 관리 활성화, MFA 삭제 보호 또는 Object Lock 설정
- **취약 기준**: 버전 관리 없음 (랜섬웨어 등으로 인한 데이터 손실 위험)

### OBJ-03. 서버 측 암호화(SSE) 적용

- **위험도**: 상
- **점검 내용**: 버킷 내 데이터 서버 측 암호화 설정 여부
- **양호 기준**: SSE-S3, SSE-KMS, SSE-C 등 서버 측 암호화 적용
- **취약 기준**: 암호화 미적용

### OBJ-04. 액세스 로그 활성화

- **위험도**: 중
- **점검 방법**:

  ```bash
  # AWS S3 예시
  aws s3api get-bucket-logging --bucket <bucket-name>
  ```

- **양호 기준**: 버킷 접근 로그 활성화, 별도 로그 저장 버킷 설정
- **취약 기준**: 접근 로그 비활성화
