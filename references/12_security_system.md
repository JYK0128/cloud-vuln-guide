## 방화벽 (Firewall)

### FW-01. 관리자 계정 및 패스워드 관리

- **위험도**: 상
- **점검 내용**: 방화벽 관리자 계정 기본 패스워드 변경 여부
- **점검 방법**:
  - 제조사 기본 관리자 계정(admin/admin, admin/password 등)으로 로그인 시도
  - 관리 콘솔에서 계정 목록 확인
- **양호 기준**: 기본 패스워드 변경, 복잡도 정책 적용
- **취약 기준**: 기본 패스워드 유지
- **조치 방법**: 관리 콘솔에서 패스워드 즉시 변경

### FW-02. 관리 인터페이스 접근 제한

- **위험도**: 상
- **점검 내용**: 방화벽 관리 인터페이스(웹 콘솔, SSH) 접근 허용 IP 제한
- **점검 방법**:
  - 관리 인터페이스 접근 ACL(허용 IP 목록) 확인
  - 인터넷 등 불특정 외부에서 관리 포트 접근 가능 여부 테스트
- **양호 기준**: 관리자 IP 대역만 관리 인터페이스 접근 허용
- **취약 기준**: 모든 IP에서 관리 접속 허용

### FW-03. 방화벽 정책 최소화 원칙 (Default Deny)

- **위험도**: 상
- **점검 내용**: 기본 정책이 "전체 거부(Default Deny)"인지 확인
- **점검 방법**:
  - 방화벽 정책 목록 확인 (마지막 규칙이 `Deny All`인지)
- **양호 기준**: 마지막 정책은 모든 트래픽 거부, 명시적 허용만 통과
- **취약 기준**: 마지막 정책이 Allow Any 또는 Default Accept

### FW-04. 불필요하거나 과도한 허용 규칙 제거

- **위험도**: 상
- **점검 내용**: `Any-to-Any`, `모든 포트 허용` 등 과도한 정책 존재 여부
- **점검 방법**:
  - 방화벽 정책 목록에서 Source, Destination, Port가 `Any`인 규칙 확인
- **양호 기준**: 업무상 필요한 특정 IP·포트만 허용
- **취약 기준**: `source=any, dest=any, port=any, action=permit` 규칙 존재

### FW-05. 정책 로그 기록 활성화

- **위험도**: 중
- **점검 내용**: 방화벽 차단/허용 이벤트 로그 기록 여부
- **점검 방법**:
  - 방화벽 정책별 로그 설정 확인
  - Syslog 전송 설정 확인
- **양호 기준**: 차단 이벤트 및 관리자 로그인 이벤트 로깅, 외부 Syslog 전송
- **취약 기준**: 로그 미기록 또는 로컬만 기록 (단기 보관)

### FW-06. 패치 및 버전 관리

- **위험도**: 상
- **점검 내용**: 방화벽 펌웨어/OS 최신 보안 패치 적용 여부
- **점검 방법**:
  - 관리 콘솔에서 현재 펌웨어 버전 확인
  - 벤더사 보안 공지 비교
- **양호 기준**: 최신 보안 패치 적용
- **취약 기준**: EOL 버전 또는 알려진 취약점이 있는 구버전 사용

### FW-07. NTP 설정

- **위험도**: 중
- **점검 내용**: 방화벽 시간 동기화(NTP) 설정 여부 (로그 신뢰성)
- **양호 기준**: NTP 서버 설정 및 정상 동기화
- **취약 기준**: NTP 미설정 (로그 타임스탬프 신뢰 불가)

## 침입방지시스템 (IPS)

### IPS-01. 시그니처 업데이트 주기

- **위험도**: 상
- **점검 내용**: IPS 탐지 시그니처 최신 버전 유지 여부
- **점검 방법**:
  - IPS 관리 콘솔에서 시그니처 버전 및 마지막 업데이트 시간 확인
- **양호 기준**: 시그니처 자동 업데이트 설정, 최근 24시간 이내 업데이트
- **취약 기준**: 장기간(1주 이상) 시그니처 미업데이트

### IPS-02. 인라인 모드 (차단 모드) 동작 확인

- **위험도**: 상
- **점검 내용**: IPS가 탐지 전용(모니터링)이 아닌 차단(인라인) 모드로 동작 여부
- **점검 방법**:
  - IPS 운영 모드 확인 (관리 콘솔 또는 네트워크 구성도)
- **양호 기준**: 인라인 모드(Inline/Active Block) 동작
- **취약 기준**: 탐지만 하고 차단하지 않는 모니터링 모드만 운영

### IPS-03. 관리 인터페이스 접근 제한

- **위험도**: 상
- **점검 내용**: IPS 관리 콘솔 접근 통제
- **양호 기준**: 전용 관리 네트워크 또는 허가된 IP만 접근 허용
- **취약 기준**: 모든 IP에서 관리 접속 허용

### IPS-04. 이벤트 로그 및 알람 설정

- **위험도**: 중
- **점검 내용**: 탐지/차단 이벤트 로그 기록 및 알람 설정
- **점검 방법**:
  - 이벤트 로그 보관 정책 확인 (기간, 외부 전송 여부)
  - 심각도 높은 이벤트 알람(이메일, SIEM) 설정 여부
- **양호 기준**: 이벤트 로그 90일 이상 보관, SIEM 또는 알람 연동
- **취약 기준**: 이벤트 로그 단기 보관 또는 알람 없음

### IPS-05. 오탐(False Positive) 관리

- **위험도**: 하
- **점검 내용**: 정상 트래픽이 차단되는 오탐 사례 검토 및 정책 조정 여부
- **양호 기준**: 정기적인 오탐 검토 및 정책 fine-tuning 수행
- **취약 기준**: 오탐 많아 IPS 기능 비활성화 또는 Pass 정책으로 우회

## 웹 방화벽 (WAF)

### WAF-01. WAF 차단 모드 동작

- **위험도**: 상
- **점검 내용**: WAF가 탐지 전용이 아닌 차단 모드로 운영 여부
- **양호 기준**: 차단 모드(Block Mode) 운영
- **취약 기준**: 탐지만 하는 모니터링 모드만 운영

### WAF-02. OWASP Top 10 방어 정책

- **위험도**: 상
- **점검 내용**: SQL Injection, XSS, CSRF, 파일 업로드, 명령어 인젝션 등 주요 공격 방어 정책 설정
- **점검 방법**:
  - WAF 관리 콘솔에서 OWASP 관련 룰 셋 활성화 여부 확인
- **양호 기준**: OWASP Top 10 대응 룰 셋 활성화
- **취약 기준**: 기본 룰 셋 비활성화 또는 최소 설정

### WAF-03. SSL 오프로딩 및 TLS 설정

- **위험도**: 상
- **점검 내용**: WAF의 SSL 오프로딩 설정 및 TLS 버전 관리
- **양호 기준**: TLS 1.2 이상, 취약 Cipher Suite 비활성화
- **취약 기준**: TLS 1.0/SSLv3 허용

### WAF-04. 로그 및 보고서 설정

- **위험도**: 중
- **점검 내용**: WAF 탐지/차단 로그 보관 및 보고서 생성
- **양호 기준**: 공격 로그 90일 이상 보관, 주기적 보고서 생성
- **취약 기준**: 로그 단기 보관 또는 미설정

### WAF-05. IP 차단 목록 관리

- **위험도**: 중
- **점검 내용**: 알려진 악성 IP 차단 목록(Blacklist) 최신화 여부
- **양호 기준**: 위협 인텔리전스 기반 IP 차단 목록 최신화
- **취약 기준**: IP 차단 목록 장기간 미업데이트

## VPN

### VPN-01. 강력한 인증 방식 적용

- **위험도**: 상
- **점검 내용**: VPN 접속 시 다중 인증(MFA/2FA) 적용 여부
- **점검 방법**:
  - VPN 게이트웨이 설정에서 인증 방식 확인
- **양호 기준**: 패스워드 + OTP/인증서 등 다중 인증 적용
- **취약 기준**: 단순 패스워드만으로 VPN 접속 허용

### VPN-02. 최신 암호화 알고리즘 사용

- **위험도**: 상
- **점검 내용**: VPN 터널 암호화 알고리즘 및 키 교환 방식 적절성
- **점검 방법**:
  - VPN 정책에서 Phase 1/2 암호화 설정 확인 (IKEv2 선호)
- **양호 기준**: AES-256, SHA-256 이상, IKEv2, PFS 활성화
- **취약 기준**: DES/3DES, MD5, IKEv1 사용

### VPN-03. 스플릿 터널 제한

- **위험도**: 중
- **점검 내용**: 스플릿 터널링(인터넷과 VPN 동시 사용) 제한 여부
- **양호 기준**: Full Tunnel 모드 사용 (모든 트래픽 VPN 통과)
- **취약 기준**: Split Tunnel 허용 (인터넷 직접 통신으로 보안 우회 가능)

### VPN-04. 세션 타임아웃 설정

- **위험도**: 중
- **점검 내용**: 유휴 VPN 세션 자동 종료 설정
- **양호 기준**: 유휴 세션 30분 이내 자동 종료
- **취약 기준**: 세션 타임아웃 미설정 (무제한 세션 유지)

### VPN-05. 로그 기록 및 모니터링

- **위험도**: 중
- **점검 내용**: VPN 접속/해제 이벤트 로그 기록 및 이상 접속 모니터링
- **양호 기준**: 접속 로그 90일 이상 보관, 이상 접속(비정상 시간대, 위치) 알람
- **취약 기준**: 로그 미기록 또는 단기 보관
