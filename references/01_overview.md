# 1장. 개요

## 1.1 개요

클라우드 취약점 점검 가이드는 클라우드 인프라 구성 요소(하이퍼바이저, 서버, 데이터베이스, 웹서버, 컨테이너, 미들웨어, 스토리지, 네트워크 장비, 정보보호시스템 등)에 대한 보안 설정 점검 기준을 제공한다.

과학기술정보통신부·한국인터넷진흥원(KISA)이 2024년 발행한 본 가이드는 클라우드 서비스 공급자(CSP) 및 클라우드 이용자가 보안 취약점을 사전에 식별하고 조치하기 위한 목적으로 작성되었다.

## 1.2 목적 및 활용

- **목적**: 클라우드 환경에서 발생할 수 있는 보안 취약점을 체계적으로 점검하고 개선 방향 제시
- **활용 대상**: 클라우드 서비스 제공자, 클라우드 이용 기업의 보안 담당자, 보안 컨설턴트
- **활용 방법**:
  1. 운영 중인 클라우드 인프라 구성 요소 목록 파악
  2. 해당 구성 요소의 점검 카테고리 확인
  3. 각 항목별 점검 절차 수행
  4. 취약 항목 조치 후 재점검

## 1.3 유의사항

1. **운영 중인 시스템 점검 시 주의**: 점검 명령어 실행 시 서비스 중단이 발생하지 않도록 사전에 영향도를 파악해야 한다.
2. **권한 확인**: 일부 점검 항목은 관리자(root/Administrator) 권한이 필요하다.
3. **버전 호환성**: 소프트웨어 버전에 따라 설정 파일 경로, 명령어, 파라미터가 다를 수 있다.
4. **환경 특수성 고려**: 본 가이드는 일반적인 기준을 제시하므로, 각 기관의 보안 정책이나 업무 특수성에 맞게 조정 적용해야 한다.
5. **법적 근거 준수**: 점검은 반드시 시스템 소유자의 승인 하에 수행해야 한다.

## 점검 항목 위험도 기준

| 위험도 | 설명 |
| --- | --- |
| **상(High)** | 즉시 조치 필요. 침해 사고 발생 가능성이 높거나 직접적인 피해가 예상되는 항목 |
| **중(Medium)** | 단기 내 조치 필요. 취약점 악용 시 상당한 피해가 발생할 수 있는 항목 |
| **하(Low)** | 중장기 개선 필요. 취약점 악용 가능성은 낮으나 보안 강화를 위해 개선이 필요한 항목 |

## 점검 결과 판정 기준

| 판정 | 설명 |
| --- | --- |
| **양호** | 점검 기준을 충족하여 보안 위협에 대한 대비가 충분한 상태 |
| **취약** | 점검 기준을 충족하지 못하여 보안 위협에 노출될 가능성이 있는 상태 |
| **해당없음** | 해당 항목이 점검 대상 시스템에 적용되지 않는 경우 |
